# Final Project

The final project of this course is you will build a **Dynamic Web Application** that acts as a portal for a specific niche. 

---

## Architectural Requirements

* **Project Pitch:** You must submit a Project Pitch in your `README.md` before coding. 
* **User Persona:** You must define the "User Persona" (who is using this?) and the "Problem" (what are they trying to do?). 
* **Layout Sketch:** Add a Low-Fidelity Sketch of your layout to your repository.
* **Research Log:** Add a research file like `NOTES.md` to capture any of your research to refer back to.

---

## Technical Requirements

### Advanced Layout and Semantics
* **Multi-View Architecture:** Implement at least three distinct views. These can be separate HTML files or a "Single Page" approach where JS toggles visibility.
* **Responsive Design:** Your CSS must handle at least one complex component that changes drastically between mobile and desktop.
* **Consistency:** Use a consistent color palette and spacing to ensure a professional, unified design system (CSS variables can be used).

### Interactivity and Data
* **Data Gallery:** Build a dynamic data gallery using a local JavaScript array. Render items as cards and enable real-time filtering based on user search input (min 6 items).
* **Live Widget:** Add a business-relevant live widget that uses the `fetch()` API to asynchronously pull data from a public source, such as weather, current time, or a daily quote, and display it in a dedicated section of the page.
* **Client-side Validation:** Build a "Functional" form with custom error messaging that doesn't rely solely on HTML5 `required` attributes.

---

## Deployment and Workflow

* **Git Story:** Minimum of 10 commits. We will be looking for **"Atomic Commits"** (one feature per commit) rather than one giant "finished project" upload. Make branches and PRs as applicable.
* **README Documentation:** You must include a section on **"Challenges & Solutions"**—describe one technical hurdle you faced and how you specifically solved it.
* **Live Production:** Deploy to a cloud provider (Vercel, Netlify, or GitHub Pages).
